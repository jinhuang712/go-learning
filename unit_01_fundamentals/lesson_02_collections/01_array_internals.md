# 1. 数组 (Array) 核心剖析

在 Go 语言中，数组是一个非常底层且严格的数据结构。虽然在实际业务代码中我们极少直接使用数组（大部分情况都在用切片），但理解数组是理解切片的前提。

## 1.1 长度是类型的一部分 (定长)
在 Java 中，`int[]` 就可以代表任意长度的整型数组。
但在 Go 中，**数组的长度是其类型签名的一部分**。
- `[3]int` 和 `[4]int` 是两种**完全不同的类型**。
- 你不能把一个 `[3]int` 的变量赋值给 `[4]int` 的变量，也不能把它们传给同一个函数（除非使用泛型或接口）。

```go
// 声明一个长度为 3 的 int 数组，默认初始化为 [0, 0, 0]
var arr1 [3]int 

// 使用字面量初始化
arr2 := [3]int{1, 2, 3}

// 让编译器自动推断长度 (注意这里是 [...] 而不是 [])
arr3 := [...]int{1, 2, 3, 4, 5}
```

## 1.2 一切皆为值传递 (Copy)
这是 Go 数组与 Java 数组最大的区别。
- **Java**: 数组是对象。当你把数组传给方法时，传递的是“对象的引用”（地址）。方法内修改数组，外部也会跟着变。
- **Go**: 数组是纯粹的**值类型**。当你把数组赋值给另一个变量，或者传递给函数时，Go 会**把整个数组的内存完整拷贝一份**。

```go
func modifyArray(a [3]int) {
    a[0] = 999 // 这只是修改了副本，不会影响外部的数组
}

arr := [3]int{1, 2, 3}
modifyArray(arr) 
// arr 依然是 [1, 2, 3]
```
**性能陷阱**: 如果你有一个 `[10000]int` 的巨型数组，传值会导致极大的内存拷贝开销。此时必须传递数组的指针 `*[10000]int`，或者更推荐的做法是——直接使用**切片 (Slice)**。
